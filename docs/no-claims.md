# Current Non-Claims

`asha-demo` is a human-facing ASHA reference playable loop, not a full native FPS.

This repo currently does not claim:

- live native RuntimeSession attach;
- Studio live inspection or control;
- interactive Three.js-rendered gameplay;
- pixel-golden rendering or production renderer polish;
- true browser pointer lock;
- local generation implementation, pathfinding, collision, combat, policy, or lifecycle authority;
- autonomous enemy movement authority; enemy movement proposals currently report `movement_authority_not_wired`;
- options or exit menu implementation; those HUD controls currently expose typed unsupported placeholders;
- demo acceptance based only on `asha-testing` synthetic proof.

The served UI may drive a reference RuntimeSession loop from public ASHA package roots: first-person generated tunnel canvas preview from renderer projection summary, movement/look with collision readout, generated tunnel readout, autonomous enemy policy proposal/tick readout, static target fire/health, HUD/menu projection controls, lifecycle death status, and typed restart receipt. That evidence proves package-root consumption and objective loop transitions; it does not prove native attach, local authority, live applied procedural dungeon gameplay, autonomous enemy movement, full menu implementation, pixel-golden rendering, or a production rendered FPS.

When one of these claims becomes real, it needs upstream ASHA/Studio public surfaces plus browser-visible evidence, not local engine substitutes.
